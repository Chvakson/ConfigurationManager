На основании тестовго задания был реализован сервис ConfigurationManager - система управления конфигурациями в web-системе.

К параметрам настроек относятся:
- Цветовая тема
- Цветовая схема(фон, тескт, акцентный цвет)
- Шрифты (семейство, размер, высота строки)
- Горячие клавиши
- Расположение окон (позиция главного окна, боковая панель)

1. API для управления конфигурациями

* Предварительно реализована логика для работы с пользователями в системе
* При создании нового пользователя, для него создается конфигурация по умолчанию из готового шаблона. Созданная конфигурация становится активной.
* При создании конфигурации для нее создается версия, которая является активной.
* При создании или обновлении конфигурации срабатывает валидация:
  - **Уникальность имени** конфигурации в рамках одного пользователя
  - **Параметров конфигурации** на основании допустимых значений из `ConfigurationConstants.cs`
* Для get /api/Configuration реализована фильтрация по названию и дате создания


2. Real-Time с использованием SignalR

* При создании/обновлении/удалении конфигурации отправляются уведомления о событии всем подписавшимся пользователям
* Поьзователи могут подписаться на все уведомления сразу, либо на отдельные
* При подписке на любые события автоматически отправляется поный список конфигураций

3. Хранение данных

* В качестве базы данных использовал SQLite.

4. Архитектура и тестируемость

* Обеспечил минимальные зависимости между компонентами (разделил на проекты/каталоги, обернул в интерфейс для всех зависимостей, подключил внедрение зависимостей, создал отдельные DTO)

5. Тестирование и документация

* Написал unit-тесты для основных операций и бизнес-логики.
* Использовал Swagger для описания API
